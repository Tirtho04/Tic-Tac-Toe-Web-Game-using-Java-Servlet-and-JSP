Class Responsibilities

GameLogic: Contains game state and business logic. Validates moves and detects winner/draw.
GameServlet: Controls the flow of the application. Handles requests and manages session.
index.jsp: View layer. Renders the UI and sends user actions to the servlet.
3. MVC ARCHITECTURE (WITHOUT DATABASE)
MODEL
GameLogic.java
  • Maintains the game state (board and player).
  • Validates and processes moves.
  • Checks winner or draw condition.
  • Provides data to controller.
VIEW
index.jsp
  • Displays the 3×3 board.
  • Shows current turn, messages and scores.
  • Sends player moves to the controller.
  • Provides New Game (reset) action.
CONTROLLER
GameServlet.java
  • Receives HTTP requests (GET/POST).
  • Invokes GameLogic methods based on user actions.
  • Stores game object in HTTP Session.
  • Uses PRG pattern (Redirect) after POST.
